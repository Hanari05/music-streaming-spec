Báo cáo: 7 Yêu cầu - Bài kiểm tra giữa kì

File index.html chứa cấu trúc báo cáo, chừa chỗ để bạn thêm ảnh, link HTML prototype và video demo.

1. Thiết kế sơ đồ Use Case

Yêu cầu: ảnh + mô tả

Mô tả: Trình bày các tác nhân (actor) và các ca sử dụng (use cases). Đính kèm hình ảnh sơ đồ và phần mô tả chi tiết cho từng use case.

Mô tả chi tiết (ví dụ)

- Actor: Người dùng — Mô tả: Người dùng có thể đăng nhập, tìm kiếm, phát nhạc.
- Use Case: Đăng nhập — Mô tả: Người dùng nhập email và mật khẩu, hệ thống xác thực và chuyển đến trang chính.

Chỉnh sửa ảnh Thêm mô tả chi tiết (tệp .md hoặc .pdf)

3. Thiết kế sơ đồ Thực thể–Kết hợp (ER)

Yêu cầu: ảnh

Mô tả: Vẽ các thực thể, thuộc tính và quan hệ. Sử dụng sơ đồ ER truyền thống để thể hiện ràng buộc chính/ngoại khoá (nếu cần).

Thay src bằng URL ảnh ER diagram.

Tải ảnh ER

4. Thiết kế sơ đồ CSDL (ERD — Logical / Physical)

Yêu cầu: ảnh

Mô tả: Cung cấp ERD dạng Logical (mô hình lôgic) và Physical (bảng, khoá, kiểu dữ liệu). Nếu có, đính kèm SQL DDL (CREATE TABLE).

Thay src bằng URL ảnh ERD (logical/physical).

Ví dụ: SQL DDL

-- Thí dụ bảng Users
CREATE TABLE Users (
  id INT PRIMARY KEY,
  email VARCHAR(255) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

6. Quy tắc nghiệp vụ & Test case

Yêu cầu: chừa link ảnh

Mô tả: Liệt kê quy tắc nghiệp vụ quan trọng (business rules) và các test case kèm theo bước thực hiện, dữ liệu đầu vào và kết quả mong đợi. Chừa chỗ để bạn đính kèm ảnh minh hoạ test (screenshot kết quả).

Ví dụ ngắn

  • BR-01: Người dùng phải đăng ký bằng email hợp lệ.
  • TC-01: Đăng ký thành công với email hợp lệ — Kết quả mong đợi: Tạo tài khoản và gửi email xác nhận.
